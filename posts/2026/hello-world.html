Hello World: Building in the Open

March 10, 2026 · 5 min read · By Ids Achterhof

meta blog architecture

TL;DR — After 30+ years building enterprise systems, I'm starting a blog to share lessons learned, architectural patterns, and thoughts on where technology is heading. This site is built with pure HTML and CSS — no frameworks, no build tools.

Why Now?

Welcome to my corner of the internet. After three decades of building enterprise systems — from mainframes to microservices, from waterfall to agile, from on-premises to cloud-native — I've accumulated a lot of lessons. Some hard-won, some surprisingly simple. All worth sharing.

I've always been someone who learns by doing. But I've realized that writing about what you've learned forces a different kind of clarity. It makes you question assumptions, refine mental models, and articulate things that have been floating around as intuition.

Good design is as little design as possible. Less, but better, because it concentrates on the essential aspects, and the products are not burdened with non-essentials.

— Dieter Rams

The Tech Stack

This site is intentionally simple. Pure HTML and CSS. No JavaScript frameworks, no build tools, no static site generators. Just the web platform as it was designed to be used.

Here's the complete setup: 

<!DOCTYPE html>
<html lang="en" data-theme="light">
  <head>
    <meta charset="utf-8" />
    <link rel="stylesheet" href="main.css" />
  </head>
  <body>
    <header><nav>...</nav></header>
    <main class="container">
      <article>Your post here</article>
    </main>
    <footer>...</footer>
  </body>
</html>

The CSS uses design tokens (CSS custom properties) for theming, follows BEM naming conventions, and styles HTML5 elements directly — minimal classes needed. Light and dark mode are built in through the data-theme attribute.

What to Expect

Topics I plan to cover:

  • Enterprise architecture patterns — the ones that survive the test of time
  • System resilience — designing for failure, graceful degradation, and recovery
  • AI and architecture — how artificial intelligence is changing how we design systems
  • Web fundamentals — HTML, CSS, and the power of the platform
  • Career reflections — lessons from 30+ years in the trenches

Posting Cadence

I aim for quality over quantity. Expect a new post every week or two — each one a self-contained piece that's worth your time to read. No filler, no fluff.

The Content Workflow

The workflow is deliberately simple:

  1. Write content in Markdown
  2. Convert to semantic HTML using the site's template
  3. Commit and push to the repository
  4. GitHub Actions deploys to GitHub Pages automatically

No CMS, no database, no server-side rendering. Just files in a repository. Version controlled, transparent, and easy to maintain.
